# AGENTS — изолированная зона Router (`experiments/router`)

## Как пользоваться этим файлом (обязательно)

- **Приоритет №1** — держать этот `AGENTS.md` актуальным: после каждого значимого шага обновляй чеклист, текущее задание и «следующее задание».
- **Не веди историю** в этом файле: только краткий контекст к цели, текущее задание, один блок «следующее задание», чеклист по текущей итерации. Старые чеклисты и длинные логи — в отдельные файлы в `tmp/` или в коммиты, не сюда.
- **Источник истины** — работоспособность, простота кода, минимум отклонений от архитектуры `wireguard-go`, где это оправдано. Документ «идеальной архитектуры» — **референс**, его можно исправлять, если реальность или код показывают ошибку, без смены главной идеи (L3 между «пирами», модуль sing-box).

---

## Цель

Получить **L3-связность между клиентами** через **универсальный практичный интерфейс Router** с хорошей архитектурой: эволюция идей `wireguard-go` (очереди, peer/route-модель, маршрутизация) **без** семантики WireGuard/Noise как протокола; эндпоинт — **часть ядра sing-box** в этой изолированной копии, код Router — **чистый, профессиональный**, на уровне публичного API внутри endpoint.

**Критерий успеха (финальный):**

1. Собрать ядро sing-box из `experiments/router/hiddify-sing-box` с **новым эндпоинтом Router** (`go build ./cmd/sing-box` — без ошибок).
2. Развернуть и проверить на **реальном VPS** из стенда в `tmp/universal-singlehop-31.56.211.60/` (тот же хост `31.56.211.60`, доступ по SSH с тем же ключом, что уже используется для этого стенда; секреты и ключи в репозиторий не коммитить).
3. Поднять **локальные Docker-клиенты**, смотрящие на этот VPS, и проверить **SMB между клиентами по VPN** (трафик через туннель/оверлей, не через bridge Docker как «LAN между контейнерами»).
4. Маршрутизация между клиентами должна опираться на **идентичность от протокола** (vless / hysteria2 / tuic / др.): имя/идентификатор пользователя сессии → Route/ACL; при неуспехе — расследование → правка → повтор теста.

---

## Краткий контекст к цели

- Рабочая изолированная копия: `experiments/router/hiddify-sing-box` + локальный `experiments/router/wireguard-go` (`replace` в `go.mod` на `../wireguard-go`).
- Router не привязан к одному протоколу: источник истины для «кто вошёл» — **user/session контекст** inbound в sing-box.
- SMB и стенд — интеграционный критерий «реальный L3», не юнит-тест только на компиляцию.
