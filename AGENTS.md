# Правила для ассистентов (репозиторий `hiddify-app`)

## Цель по `l3router` (целевая модель)

Нужен **L3-хаб на VPS (звезда)**: статические маршруты как **пиры WireGuard** (фиксированные префиксы в конфиге), форвардинг по **dst/src** в IP-пакетах и привязке сессии к маршруту, **без mesh** и **без обязательного runtime control plane**. Криптография остаётся в транспорте sing-box, не в движке L3.

**Идеальная спецификация и архитектурные детали:** `experiments/router/docs/router-architecture.md`.

## Текущее состояние кода (зафиксировать явно)

Реализованный в дереве **l3router** выполнен **неверно относительно этой цели**: ориентация на **control plane** (owner, runtime upsert маршрутов через API, тяжёлые стенды). Это **вариант с control plane**, а не целевая «звезда как WG».

Пока архитектура не приведена к документу выше, считать результат **неудовлетворительным** для продукта.

## Критерий проверки данных (стенд)

Для объективной проверки связности по-прежнему использовать **реальную передачу ≥100 МБ** (например SMB) с контролем целостности; частичные пробы без полезной нагрузки недостаточны.

## Локальный стенд `universal`

Каталог `stand-multi-cascad-prod-l3-universal/` — локально для Docker, **не коммитится** (см. `.gitignore`).

## Стенд на VPS (`31.56.211.60`)

Каталог `tmp/universal-singlehop-31.56.211.60/` (часто в `.gitignore`): VLESS REALITY + l3router, Docker-клиенты. Скрипты и артефакты там отражают **текущий** (control-plane) стенд, не целевую минимальную эксплуатацию.
